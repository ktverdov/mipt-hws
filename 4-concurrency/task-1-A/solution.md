Измененный таким образом алгоритм Петерсона для двух потоков не гарантирует взаимное исключение, 
т.к. оба потока одновременно могут находиться в критической секции.

Пример, вызывающий такое поведение:
-Потоки почти одновременно вызывают lock():
	Thread_0					Thread_1
	victim = 0
							victim = 1
							want[1] = true
						(поток входит в критическую секцию кода, 
						т.к. условие цикла ожидания не выполнено - 
						want[0] = false)
	want[0] = true
(поток входит в критическую секцию,
т.к. условие цикла ожидания не выполнено - 
victim == 1)
